Shun Knives App Design

This UI concept started with exploration of Shun brand. Althought they seem to market their knives as premium japanese products, their brand identity didn’t really reflect that vibe.

Since there was no smartphone app, I figured I should try to create one to practise my UX/UI skills by solving hypothetical user problems.

Issues

Official website provides user with different categories and filters to choose from.

I argued, that it’s overwhelming to select one for those who don’t know much about chief’s knifes since there is no visual representation.

    Most of the UX decisions were build around an assumption, that it’s hard to choose the right knife, especially for an inexperienced cook.

    To fix that, I’ve set up following goals:

  • Minimize search efforts.

  • Make UI more premium and authentic.

  • Minimize interaction friction with product to maximize sales.

Typography
&
Colors

To direct user’s attention on the product itself rather than UI, I’ve chosen neutral snow white as primary color and variety of grey shades for the secondary elements - such pallete insures app’s compatability with the dark theme.

Frosty-red gradient emphasizes actionable elements and theirs’ interactive states.

Most european cultures interpret white and it’s descendents as sigils of purity and holiness; with red, which means power and blood, this pallete emphasizes the japanese aestethics and premiality of the knives.

In combination with the pallete, Museo Sans’s elegance and straightness reasembless japanese glyphs. It’s sans-serif nature made product names stand out in the most exquisite way.

Bold is used to emphasize the most conversionable text elements.

All texts are sized and optimized based on Material typography guidance, assuring it’s readability.

Information Architecture

I’ve prepared a screen map to have an overview of all the screens I need to design and how they are related to each other.

Thanks to that, I was able to quickly find appropriate components and build the app without any compromises between functionality & design.

Components

After understanding how my app is going to function, I decided to use layers in my design.

By sketching different ideas, I’ve created components which formed the front layer.

In order to control what is shown on front layer, I’ve created different filters based on available parameters of the product.

Interactions

On start up, app greets user with brand logo and card animations, implicitly “teaching” user by showing swipe directions for all avaliable actions.

By default, frontlayer is set to discover mode - a card view, which shows user most relevant knives based on active filters and category.

By swiping downcard, user can check wether there is a different size knife avaliable.

Backlayer consists of filters and categories, which control and inform what content to display on the front layer.

This way, user sees the examples of applied filters with the sample of products.

Swiping up a product card reveals description, gallery and reviews about the knife.

Cart sheet header shows number of added items and is one swipe away from the checkout.
